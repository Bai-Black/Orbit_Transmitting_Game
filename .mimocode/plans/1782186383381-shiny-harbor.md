# Plan: 轨道预测与入轨判定重构

## 概述
重构入轨成功判定逻辑，增加轨道预测可视化和近远地点标注。

## 修改文件
- `index.html`（唯一文件）

## 新需求

### 1. 轨道预测（当飞行器在合理轨道范围时）
- 当飞行器距离天体 B 在 `[radius+50, radius+400]` 范围内时，启动轨道预测
- 基于当前状态（位置+速度），用数值积分向前模拟 ~2 个轨道周期
- 预测轨迹用**虚线**绘制在画面上

### 2. 轨道闭合检测
- 检查预测轨迹末端是否回到起点附近（距离 < 轨道半径的 20%）
- 若闭合 → 标注近地点/远地点
- 若不闭合 → 只显示预测轨迹虚线，不标注

### 3. 近地点/远地点标注
- 在预测轨迹上找到距天体 B 最近点（近地点）和最远点（远地点）
- 在两点处标注高度值
- 若两点差 < 轨道平均半径的 10% → 视为圆轨道，只在轨道旁标注平均高度

### 4. 入轨判定（新条件）
- 轨道闭合（预测轨迹形成闭合环）
- 近地点高度 > `ORBIT_MIN`（天体半径 + 60）
- 远地点高度 < `ORBIT_MAX`（天体半径 + 300）
- 两个条件同时满足时开始计时

### 5. 胜利条件
- 上述入轨条件满足后，飞行器完成**一整圈**（回到起始角度附近）且状态未变化 → 胜利
- 计时方式：记录满足条件时的角度，检测角度变化累计 ≥ 2π

## 实现步骤

### 步骤 1：轨道预测函数 `predictOrbit()`
- 输入：飞行器当前位置/速度，主导天体
- 用欧拉积分向前模拟，步长 FIXED_DT，模拟时长约 2 个周期
- 返回：预测轨迹点数组 `[{x, y}]`

### 步骤 2：轨道分析函数 `analyzeOrbit(points, body)`
- 输入：预测轨迹点，主导天体
- 计算每个点到天体中心的距离
- 找到最小距离点（近地点）和最大距离点（远地点）
- 检测闭合性：首尾距离 vs 平均轨道半径
- 计算近远地点差，判断是否为圆轨道
- 返回：`{ closed, periapsis, apoapsis, periAlt, apoAlt, isCircular, avgAlt }`

### 步骤 3：绘制预测轨迹 `drawOrbitPrediction()`
- 只在飞行器处于天体 B 引力范围且距离合理时显示
- 虚线绘制预测轨迹
- 标注近地点/远地点（非圆轨道时）
- 圆轨道时标注平均高度

### 步骤 4：重构入轨判定逻辑
- 替换原有的简单距离+速度判定
- 新条件：闭合 + 近地点高度 + 远地点高度
- 满足条件时开始轨道周期计数
- 完成一圈 → WIN

### 步骤 5：清理
- 移除旧的 `winTimer` 逻辑和 `drawShip` 中的虚线圆弧
- 更新 HUD 显示轨道状态信息

## 验证
1. 在天体 A 轨道上 → 不显示预测轨迹
2. 飞向天体 B 进入范围 → 显示预测虚线
3. 椭圆轨道 → 显示近地点/远地点标注
4. 圆轨道 → 显示平均高度
5. 闭合+近远地点在范围内 → 开始计圈
6. 完成一圈 → 胜利
