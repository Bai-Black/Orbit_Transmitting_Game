# Plan: 2D 轨道转移网页游戏

## 概述
创建一个单文件 HTML5 Canvas 轨道力学游戏。飞行器初始绕一个天体运行，玩家通过姿态旋转和油门控制，将其变轨至另一个天体。视觉风格：简约黑白像素风（类似 Chrome 恐龙小游戏）。

## 文件
- `index.html` — 单文件，包含所有 HTML/CSS/JavaScript，无外部依赖

## 视觉风格
- **配色**：黑底白线（#000 背景，#fff / #ccc 线条和文字）
- **天体**：用白色圆环（描边）表示，中心可有简单十字标记；两个天体大小不同以区分
- **飞行器**：小三角形（白色填充），用 ASCII/像素风格
- **油门火焰**：简单的锯齿线条或短横线表示推力方向
- **星星背景**：随机散布的白色小点（单像素）
- **HUD**：左上角像素字体显示速度、距离、油门等信息
- **轨迹**：白色小点连线显示飞行路径
- **整体风格**：无渐变、无阴影、无彩色，纯黑白二值

## 物理引擎

### 引力
- `F = GM/r²`，飞行器同时受两个天体引力
- 牛顿第二定律：`a = F/m`（飞行器质量约掉）
- 对两个天体分别计算引力加速度，矢量叠加

### 推力
- 上箭头：增大油门（0→100%），下箭头：减小油门
- 推力加速度方向 = 飞行器朝向方向
- `a_thrust = throttle * thrustMax * (cos(angle), sin(angle))`

### 旋转
- 左箭头：逆时针旋转，右箭头：顺时针旋转
- 仅改变飞行器朝向角度，不直接影响速度

### 数值积分
- 半隐式欧拉法：先更新速度，再更新位置
- 固定时间步长 dt = 1/60s

### 关键常量
| 常量 | 值 | 说明 |
|------|------|------|
| `G` | 500 | 引力常数（屏幕尺度调优） |
| `body1.mass` | 5000 | 起始天体质量 |
| `body2.mass` | 3000 | 目标天体质量 |
| `body1.radius` | 35 | 起始天体视觉半径 |
| `body2.radius` | 25 | 目标天体视觉半径 |
| `body1.pos` | (300, 400) | 起始天体位置 |
| `body2.pos` | (1200, 400) | 目标天体位置 |
| `thrustMax` | 200 | 最大推力加速度 |
| `rotSpeed` | 2.5 rad/s | 旋转角速度 |
| `trailLength` | 200 | 轨迹点数量 |

## 游戏状态

### 1. TITLE（标题屏）
- 显示"轨道转移"大标题
- 简要操作说明（中文）
- "按空格键开始"
- 像素风格边框装饰

### 2. PLAYING（游戏中）
- 物理模拟 + 渲染
- HUD 显示信息

### 3. WIN（胜利）
- 检测到稳定绕目标天体轨道
- 显示"任务完成"
- "按空格键重新开始"

### 4. CRASH（坠毁）
- 飞行器距天体表面 < 安全距离
- 显示"坠毁"
- "按空格键重新开始"

## 初始轨道设置
- 飞行器在 Body 1 的圆轨道上
- 轨道半径 r = body1.radius + 120
- 圆轨道速度 v = sqrt(G * M / r)，方向垂直于半径矢量
- 朝向指向速度方向

## 胜利条件
- 距离 Body 2 在 `[body2.radius + 80, body2.radius + 250]` 范围内
- 飞行器速度接近该距离的圆轨道速度（±30%）
- 满足条件持续 2 秒 → 判定为稳定轨道 → 胜利

## 碰撞检测
- 距任一天体 < 该天体半径 → 坠毁
- 距离两体都 > 3000px → 逃逸，自动重置

## 摄像机
- 以飞行器为中心，`ctx.translate(-camX, -camY)` 实现世界坐标渲染
- 可平滑跟随或直接跟随

## 实现步骤

1. **HTML 骨架**：Canvas 全屏，无滚动条，黑色背景
2. **游戏循环**：requestAnimationFrame + 固定 dt 累积
3. **输入系统**：keydown/keyup 按键追踪
4. **物理引擎**：引力计算、推力、旋转、欧拉积分
5. **天体对象**：两个天体的数据结构（位置、质量、半径）
6. **飞行器**：位置、速度、角度、油门
7. **摄像机**：跟随飞行器
8. **渲染**：背景 → 星星 → 轨迹 → 天体（圆环）→ 飞行器 → 推力火焰 → HUD
9. **碰撞/胜利检测**：距离判断 + 持续时间计时器
10. **游戏状态**：标题屏、游戏中、胜利、坠毁的覆盖层
11. **调优**：调整常量使轨道运行视觉效果舒适

## 验证方法
1. 浏览器打开 `index.html`
2. 标题屏应显示中文标题和操作说明
3. 按空格开始 → 飞行器稳定绕 Body 1 运行
4. 上键增大油门 + 左右键旋转 → 观察轨迹变化
5. 成功飞到 Body 2 并进入轨道 → 胜利画面
6. 撞击天体 → 坠毁画面
7. 任意结束状态按空格重新开始
